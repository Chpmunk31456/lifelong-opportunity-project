# Accessibility

Accessibility is a core requirement of the Lifelong Opportunity Guides project.

## Goals

Guides and repository documents should:

- Use plain language and define specialized terms
- Follow a logical heading structure
- Use meaningful link text
- Avoid using color alone to communicate meaning
- Use readable fonts, spacing, and contrast
- Give tables clear headers and simple structures
- Include alternative text for informative images
- Maintain logical keyboard navigation and reading order
- Use tagged PDFs and bookmarks when supported
- Avoid unnecessary animation, flashing content, and decorative clutter

## Review Workflow

For each source document:

1. Review headings, lists, tables, links, images, and reading order.
2. Run the accessibility checker in the authoring application.
3. Test keyboard navigation.
4. Test a representative sample with screen-reader or read-aloud software.
5. Correct the editable source first.
6. Regenerate the PDF with tags and bookmarks when supported.
7. Confirm that text remains searchable and selectable.
8. Record material corrections in the revision log.

Automated checks are helpful but do not replace testing by people with disabilities or users of assistive technology.

## Reporting a Barrier

Open a GitHub issue and include:

- Guide number and title
- Language and version
- File type
- Page or section
- Assistive technology or device, if relevant
- Description of the barrier
- Suggested correction, if known

Do not include private medical information.

## Alternative Formats

Requests for alternative formats are welcome. This is a volunteer-led project, so fulfillment cannot be guaranteed, but documented accessibility needs will inform future revisions.
